# Quick Start: Extract Any Screen to Figma

## One Command Solution

```bash
# Extract sign-in screen
npm run extract-screen "app/(auth)/sign-in.tsx"

# Extract sign-up screen  
npm run extract-screen "app/(auth)/sign-up.tsx"

# Extract any screen with custom output name
npm run extract-screen "app/(tabs)/index.tsx" -- --output figma-plugin/home-screen.json
```

## What Was Fixed

### ✅ Auto-Layout Issues
- **Before**: Fixed dimensions on Container (`width: 390, height: 844`)
- **After**: Uses `flex: 1` for proper auto-layout
- **Before**: Fixed `height: 50` on inputs
- **After**: Uses `minHeight: 50` with `flexDirection: "row"`
- **Result**: Elements now properly accommodate within mobile frame with auto-layout

### ✅ SVG Icons
- **Before**: Placeholder Views with background colors
- **After**: Actual SVG data embedded in JSON
- **Type**: Changed from `View` to `Icon` with `svgData` property
- **Result**: Editable vector graphics in Figma

### ✅ Mobile Frame Size
- **Standard**: 390x844 (iPhone 14/15 size)
- **SafeAreaView**: Properly constrained at root
- **Container**: Uses `flex: 1` to fill available space

## Import to Figma

1. **Copy JSON content**:
   ```bash
   # Mac
   cat figma-plugin/sign-in-screen.json | pbcopy
   
   # Or just open the file and copy
   ```

2. **Open Figma Desktop**

3. **Run plugin**: Plugins → Code to Figma

4. **Paste and Import** ✅

## Differences: Manual vs Auto

### Manual JSON (sign-in-screen.json)
- ✅ Hand-crafted with exact specifications
- ✅ SVG icons embedded
- ✅ Uses `minHeight` for inputs
- ✅ Proper auto-layout with `flex: 1`

### Auto-Generated JSON (via script)
- ✅ Extracts from actual React Native code
- ✅ Automatically detects social auth buttons

- ⚠️ May need manual adjustments for complex layouts

## Recommended Workflow

1. **Use the script for initial extraction**:
   ```bash
   npm run extract-screen "app/(auth)/screen.tsx"
   ```

2. **Review and adjust the JSON** if needed

3. **Import to Figma**

4. **Iterate**: Make changes in code → Re-extract → Re-import

## Key Best Practices

✅ **Always use**:
- `flex: 1` on containers
- `minHeight` instead of fixed `height`
- `flexDirection: "column"` explicitly
- `justifyContent: "space-between"` for layouts
- SVG `Icon` type for vector graphics
- Font weights as strings: "400", "500", "600"

❌ **Never use**:
- Fixed dimensions on flex containers
- `Signika_400Regular` font syntax
- View placeholders for icons
- Missing `flexDirection` on containers
